# EssentialsMiniReload 1.1.0-1.20.6-HIGHER-RELEASE

Copy-ready SpigotMC resource update notes.

## Changes

- Added `/emhelp`, a full command help GUI for EssentialsMiniReload.
- Added `/essentialshelp` as an alias for the new help GUI.
- Added `/emhelp text` for console users or players who prefer chat-based help.
- The help GUI now lists all plugin commands in pages.
- Hovering over a command item shows the command description, usage, aliases, permission, and extra notes where available.
- Clicking a command item sends detailed chat help and suggests the command in chat.
- Improved ProtocolLib skin refresh for `/nick <nickname> <skin>`.
- Nick skin changes now refresh the visible player entity more reliably, not only the tab-list profile.
- `/nick reset` now restores original skin data more reliably.
- Skin refresh now respects vanish visibility and should not reveal vanished players.
- Added Geyser-Spigot and Floodgate as optional soft dependencies.
- Expanded Geyser/Floodgate integration helpers for Bedrock player detection and player metadata.
- Updated `/flyspeed` with `reset`, `default`, and `normal` arguments to restore the normal fly speed.
- Added tab completion for `/flyspeed` speed values and optional player targets.
- Improved `/flyspeed` permission and usage messages to use the plugin's localized message helpers.
- Fixed the help GUI inventory holder so it no longer returns a null inventory.
- Improved location existence checks so deleted placeholder values and blank strings are handled safely.
- Added bundled EssentialsMini icon assets.
- Updated the project version to `1.1.0-1.20.6-HIGHER-RELEASE`.
- Updated `README.md` and `spigotmc-description.txt` for the new release.
- Updated `versions.json` with the 1.1.0 release description.

## Notes

- `/help` is not used, because Bukkit/Spigot already owns that command.
- Use `/emhelp` or `/essentialshelp` instead.
- ProtocolLib is still required for live skin refresh features.
