[B][SIZE=5]EssentialsMiniReload 1.1.0-1.20.6-HIGHER-RELEASE[/SIZE][/B]

[B]Resource update title[/B]
[CODE]EssentialsMiniReload 1.1.0 - Help GUI, skin refresh, economy fixes, and safer updates[/CODE]

[B]Update post[/B]
This update focuses on making EssentialsMiniReload easier to use, more reliable on modern servers, and clearer for administrators. It adds a full in-game command help GUI, improves ProtocolLib skin refresh behavior, hardens economy commands, and fixes the automatic update/download flow.

[B]New help command[/B]
[LIST]
[*]Added [B]/emhelp[/B], a full command help GUI for EssentialsMiniReload.
[*]Added [B]/essentialshelp[/B] as an alias for the new help GUI.
[*]Added [B]/emhelp text[/B] for console users or players who prefer chat-based help.
[*]The help GUI now lists all plugin commands in pages.
[*]Hovering over a command item shows the command description, usage, aliases, permission, and extra notes where available.
[*]Clicking a command item sends detailed chat help and suggests the command in chat.
[/LIST]

[B]Important command note[/B]
[LIST]
[*][B]/help[/B] is not used because Bukkit/Spigot already owns that command.
[*]Use [B]/emhelp[/B] or [B]/essentialshelp[/B] for EssentialsMiniReload help.
[/LIST]

[B]Skin and nickname improvements[/B]
[LIST]
[*]Improved ProtocolLib skin refresh for [B]/nick <nickname> <skin>[/B].
[*]Nick skin changes now refresh the visible player entity more reliably, not only the tab-list profile.
[*][B]/nick reset[/B] now restores original skin data more reliably.
[*]Fixed a [B]/nick[/B] skin refresh crash on newer ProtocolLib/Purpur builds where ProtocolLib cannot expose GameProfile properties through its wrapper.
[*]Display-name-only player info packets are no longer rewritten by the skin refresh listener, preventing errors when using [B]setPlayerListName[/B].
[*]Added a fallback skin refresh path for Java 26/Purpur builds where ProtocolLib cannot mutate final player-info packet action fields.
[*]Skin refresh now respects vanish visibility and should not reveal vanished players.
[/LIST]

[B]Economy fixes[/B]
[LIST]
[*]Fixed [B]/eco set[/B] for other players by accepting both command orders:
[/LIST]
[CODE]/eco set <amount> <player>
/eco set <player> <amount>[/CODE]
[LIST]
[*]Added tab completion for [B]/eco[/B] actions, amount suggestions, and online player targets.
[*]Hardened [B]/pay[/B], [B]/eco[/B], and [B]/bank[/B] so failed Vault transactions no longer report success.
[*][B]/pay[/B] now accepts both command orders:
[/LIST]
[CODE]/pay <amount> <player>
/pay <player> <amount>[/CODE]
[LIST]
[*]Fixed [B]/pay **[/B] so it checks the total required balance before paying other online players.
[*]Added separate [B]essentialsmini.eco.remove[/B] and [B]essentialsmini.eco.remove.others[/B] permissions for [B]/eco remove[/B].
[*]Improved bank deposit, withdraw, and transfer handling so money is rolled back if the second side of the transaction fails.
[/LIST]

[B]Command and player utility improvements[/B]
[LIST]
[*]Updated [B]/flyspeed[/B] with [B]reset[/B], [B]default[/B], and [B]normal[/B] arguments to restore the normal fly speed.
[*]Added tab completion for [B]/flyspeed[/B] speed values and optional player targets.
[*]Improved [B]/flyspeed[/B] permission and usage messages to use the plugin's localized message helpers.
[*]Fixed the help GUI inventory holder so it no longer returns a null inventory.
[*]Improved location existence checks so deleted placeholder values and blank strings are handled safely.
[/LIST]

[B]Update checker and auto-download fixes[/B]
[LIST]
[*]Fixed the update checker so 1.20.6 higher releases use the correct [B]1.20.6-higher-release[/B] metadata channel.
[*]Fixed auto-download so update jars are written into the real [B]plugins/update[/B] folder.
[*]Auto-download now writes to a temporary file first, verifies it is not empty, and then moves it into the update folder.
[*]Added config comments explaining that [B]AutoDownload[/B] requires [B]checkForUpdates[/B].
[*]Improved update-check messages for metadata failures, available updates, and failed downloads.
[/LIST]

[B]Compatibility and integrations[/B]

[*]Expanded Geyser/Floodgate integration helpers for Bedrock player detection and player metadata.
