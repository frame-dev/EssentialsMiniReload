[B][SIZE=5]EssentialsMiniReload 1.1.1-1.20.6-HIGHER-RELEASE[/SIZE][/B]

[B]Resource update title[/B]
[CODE]EssentialsMiniReload 1.1.1 - SQL table fixes and safer /nick skin refresh[/CODE]

[B]Update post[/B]
This update focuses on stability fixes for SQL/SQLite storage and the [B]/nick[/B] skin refresh on newer Purpur and ProtocolLib builds.

[B]SQL and SQLite fixes[/B]
[LIST]
[*]Fixed the missing [B]essentialsmini_ban_mute[/B] table error for SQL and SQLite storage.
[*]The ban/mute manager now creates the SQL table synchronously before login/chat checks can query it.
[*]Ban/mute SQL lookups now fall back safely instead of throwing chat/login stack traces when storage is unavailable.
[*]Reduced SQL connection log spam by removing repeated successful connection messages.
[/LIST]

[B]/nick skin refresh fixes[/B]
[LIST]
[*]Fixed a [B]/nick[/B] skin refresh crash on newer ProtocolLib/Purpur builds where ProtocolLib cannot expose GameProfile properties through its wrapper.
[*]Added a fallback skin refresh path for Java 26/Purpur builds where ProtocolLib cannot mutate final player-info packet action fields.
[*]Skin refresh now avoids rewriting final player-info packet fields directly, preventing the [B]ClientboundPlayerInfoUpdatePacket.actions[/B] warning on Java 26/Purpur.
[*][B]/nick <nickname> <skin>[/B] now writes the fetched skin texture to the live player profile before refreshing viewers, so the selected skin is actually applied instead of only changing the nickname.
[*]Skin writes now fall back from Bukkit profile objects to the NMS GameProfile when Bukkit does not expose Mojang texture properties.
[*]Skin property reflection now supports newer authlib accessors, private property-map fields, and inherited Multimap methods used by modern Purpur/Java builds.
[*]Mojang [CODE]PropertyMap[/CODE] texture updates now use the Guava [CODE]Multimap[/CODE] API directly, fixing failed texture insert/remove operations on newer authlib builds.
[*]Live skin writes now prefer the NMS player GameProfile before Bukkit profile copies and handle immutable property maps without aborting [B]/nick[/B].
[*]Added a Paper/Purpur [CODE]PlayerProfile[/CODE] runtime path for servers that expose [CODE]setPlayerProfile[/CODE].
[*]Added Spigot/NMS fallbacks that mutate the backing authlib property map or replace the player's GameProfile when the public property map is immutable.
[*]Skin refresh no longer sends remove/add tab-list packets to the nicked player, preventing empty tab lists and signed chat validation errors.
[*]Added [CODE]skinDebug: true[/CODE] diagnostics for [B]/nick[/B] skin lookup, live profile texture writes, ProtocolLib packet choices, packet sends, and entity refreshes.
[*]Display-name-only player info packets are no longer rewritten by the skin refresh listener, preventing errors when using [B]setPlayerListName[/B].
[/LIST]

[B]Configuration fixes[/B]
[LIST]
[*][CODE]config.yml[/CODE] now loads bundled defaults correctly on startup and [CODE]/essentialsmini reload[/CODE].
[*]Existing config files now receive newly added default options without overwriting administrator-set values.
[*]Bundled comments and inline comments are copied into [CODE]config.yml[/CODE] when defaults are merged.
[*]The plugin no longer immediately rewrites [CODE]config.yml[/CODE] with [CODE]saveConfig()[/CODE] during startup unless defaults or comments actually changed.
[/LIST]

[B]Compatibility notes[/B]
[LIST]
[*]This release targets the 1.20.6 higher release channel.
[*]ProtocolLib is still required for live [B]/nick[/B] skin refresh features.
[*]Servers on newer Java/Purpur builds should no longer see the skin refresh warnings from the previous build.
[/LIST]
